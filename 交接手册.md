## 项目概览

`info-hub` 是一个 **交易工作台**，核心定位：

- **交易复盘** — 基于 `uwillberich` 方法论输出复盘
- **转强选股** — 盘前生成转强候选池，次日验证闭环
- **公众号搜索** — 微信公众号文章搜索与浏览（v3.0 新增）

决策顺序：市场分类 → 概率情景 → 时间门纪律 → 证据补强

四条铁律：
1. 先定方法论，再调工具
2. 数据为辅，逻辑为主
3. 不因数据好看改变市场判断
4. tool 与 `uwillberich` 冲突时，信 `uwillberich`

## 启动方式

### 后端

```bash
cd backend
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt

# 开发模式
uvicorn main:app --reload --port 8000

# 生产模式
./start.sh start
./start.sh status
```

## 核心模块说明

### 1. 公众号搜索 (v3.0 新增)

**功能**: 搜索和浏览微信公众号文章，获取交易复盘和市场分析资讯

**架构**:
```
用户搜索 → API路由 → CRUD服务 → SQLite
                      ↓
              定时任务 (每2小时)
                      ↓
              Jina Reader 爬虫 → 搜狗微信/雪球
```

**关键文件**:
- `backend/services/wechat_crawler.py` — 爬虫（多平台搜索、去重、时间解析）
- `backend/services/wechat_crud.py` — 数据库操作（搜索、分页、统计）
- `backend/routers/wechat.py` — API 路由（8个端点）
- `frontend/src/components/wechat-search/WechatSearchPanel.tsx` — 前端页面

**API 端点**:
| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/wechat/search` | 搜索文章（支持关键词+分类+分页） |
| GET | `/api/wechat/accounts/{id}/articles` | 公众号文章列表 |
| GET | `/api/wechat/accounts/{id}` | 公众号详情 |
| GET | `/api/wechat/trending-topics` | 热门话题 |
| GET | `/api/wechat/recommended-accounts` | 推荐公众号 |
| GET | `/api/wechat/categories` | 分类列表 |
| GET | `/api/wechat/statistics` | 统计信息 |
| POST | `/api/wechat/cleanup` | 清理旧文章 |

### 2. 复盘大师 (ReAct Agent)

- 支持普通模式和 ReAct 模式
- ReAct 模式下 LLM 可自主调用工具获取实时数据
- 7个内置工具：行情、K线、板块、历史复盘、转强候选、股票搜索
- 安全约束层防御 prompt injection 攻击

### 3. 转强选股

- 盘前生成转强候选池
- 候选分层：buy / watch / avoid
- 次日验证闭环 + 失败样本归因
- 高频历史接口短 TTL 缓存

## 注意事项

1. **Jina Reader** 用于公众号搜索，免费无需 API Key
2. **SQLite** 数据库位于 `~/.info-hub/` 目录
3. **日志** 位于 `~/.info-hub/logs/info-hub.log`
4. **前端主包** 有 500kB+ 告警，但不影响使用
5. **缓存** 是单进程内存缓存，适合本地/单实例场景
6. 定时任务采集的公众号文章默认保留 30 天，超期自动清理
