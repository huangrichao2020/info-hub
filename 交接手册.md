## 一、项目概览

`info-hub` 是一个 **A股全栈交易智能工作台**，核心定位为 `uwillberich` 方法论驱动的自动化交易决策系统。

### 核心功能

| 功能 | 说明 | 自动化程度 |
|------|------|-----------|
| 🔥 转强选股 | 盘前09:28筹码结构由弱转强扫描 | ✅ 自动 |
| 📊 盘前报告 | 09:00 情绪周期+风险+关注方向 | ✅ 自动 |
| ⚡ 盘中快报 | 09:35-15:00 每5分钟信号变化+预警 | ✅ 自动 |
| 📈 盘后复盘 | 15:30 涨停深度分析+主线识别+明日预判 | ✅ 自动 |
| 🧘 住相信号 | 5信号市场情绪量化监控 | ✅ 自动 |
| 🧠 复盘大师 | ReAct Agent 多工具智能对话 | 🔘 按需 |
| 📰 财经新闻 | 多源财经新闻聚合 | ✅ 自动 |
| 📱 公众号搜索 | 微信公众号文章搜索与采集 | ✅ 自动 |

### 核心理念

> "市场是情绪的集合，技术是情绪的载体。不预测，只跟随；不博弈，只确认。"

**决策顺序**（不可跳跃）：市场分类 → 概率情景 → 时间门纪律 → 证据补强

**四条铁律**：
1. 先定方法论，再调工具
2. 数据为辅，逻辑为主
3. 不因数据好看改变市场判断
4. tool 与 `uwillberich` 冲突时，信 `uwillberich`

## 七、决策引擎详解（v3.1 新增）

### 自动化流水线

```
交易日 02:00 → 数据同步（A股全量历史数据）
交易日 03:00 → 市场缓存更新（板块/涨停/指数）
交易日 09:00 → 📊 盘前作战报告
交易日 09:28 → 🔥 转强选股生成
交易日 09:35-15:00 → ⚡ 盘中快报（每5分钟）
交易日 15:30 → 📈 盘后复盘报告
```

### 报告类型

| 报告 | 内容 | 触发 |
|------|------|------|
| 盘前报告 | 情绪周期判定 + 风险评估 + 昨日涨停分布 + 今日关注方向 + 操作策略 + 方法论提示 | 自动 09:00 |
| 盘中快报 | 情绪/涨停/连板/住相/风险速览 + 预警信号 + 仓位建议 | 自动 每5分钟 |
| 盘后复盘 | 市场全景 + 涨停深度分析(Top10行业) + 主线识别 + 住相信号详情 + 明日两种剧本 + 方法论沉淀 | 自动 15:30 |

### API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/decision/pre-market` | 手动触发盘前报告 |
| GET | `/api/decision/intraday` | 手动触发盘中快报 |
| GET | `/api/decision/post-market` | 手动触发盘后复盘 |
| GET | `/api/decision/latest?report_type=pre_market` | 获取最新报告 |
| GET | `/api/decision/reports?report_type=post_market&days=7` | 历史报告列表 |
| GET | `/api/decision/status` | 决策引擎状态 |

### 方法论常量

决策引擎内置了完整的交易方法论参数（来自 Stock-Trading-Wisdom）：

- **情绪周期阈值**：冰点(<20涨停/<3板) → 复苏(30-50/3-4板) → 高潮(>80/>7板) → 退潮
- **风控参数**：单票≤15%、同题材≤2只、总仓位≤80%、止损-8%/-5%
- **住相操作映射**：0信号→观察 / 1→盯盘 / 2→减仓 / 3→必须减仓 / 4→清仓 / 5→远离
- **板块梯队**：T0情绪龙头 / T1核心中军 / T2强势跟涨 / T3补涨龙

## 八、核心系统详解

### 1. 转强选股（Turn Strong）

**核心理念**：盘前09:26集合竞价后，筛选筹码结构由弱转强的股票。

**三层筛选**（阶梯式回退）：
- MX API层：筹码获利比例变化 <60%→>60%（严格）/ <70%→>55%（宽松）
- 问财验证层：高开>0% + 竞价量比排序
- 候选合并：按竞价量比降序 → 竞价涨幅降序

**5种盘中承接状态**：转强扩散 → 强势承接 → 冲高回落 → 转弱 → 待观察

**次日验证闭环**（5级）：success(≥5%) → success(≥2%) → flat → weak(<0%) → fail(≤-2%)

### 2. 住相信号（Obsession Phase）

基于佛学概念的市场情绪框架，5个破裂信号：

| 信号 | 条件 | 阈值 |
|------|------|------|
| 龙头乏力 | 涨停家数/炸板率 | <30家 / >40% |
| 跟风先跑 | 板块涨跌比 | <0.5 |
| 扩散停止 | 热门板块/集中度 | <5个 / >70% |
| 情绪背离 | 指数涨但涨停少 | <30家 |
| 资金转向 | 主力净流出 | >100亿 |

### 3. 交叉验证（Cross Validation）

5个独立视角并行分析：
- **供需**：成交量+北向+涨跌停比
- **执念**：情绪周期+连板高度+溢价
- **住相**：K线形态+均线排列+量价
- **龙头**：主线确认+涨停数+梯队
- **宏观**：政策+美股+汇率

共识强度≥80%→强烈看多/看空，<60%→观望。

### 4. 缠论技术分析

日K缠论结构：分型检测 → 笔构建 → 线段 → 中枢 → 买卖点（一买/二买/类二买/一卖/二卖）

### 5. 复盘大师（ReAct Agent）

7个内置工具：个股行情 / 大盘指数 / K线多周期 / 领涨领跌板块 / 历史复盘 / 转强候选 / 股票搜索

## 十四、注意事项

1. **沙箱 vs macOS**：config.py 已适配两种环境，`UWILLBERICH_SKILL_ROOT` 在沙箱中指向 `~/.uwillberich/`
2. **MX API配额**：每个Key每日20次，系统自动轮转多个Key
3. **数据时效**：涨停数据基于 Baostock 缓存（300秒TTL），非完全实时
4. **盘中快报**：仅在9:30-15:00之间实际执行（scheduler中做了时段判断）
5. **数据库**：SQLite单文件，位于 `~/.info-hub/info_hub.sqlite3`，无需额外配置
6. **日志轮转**：单文件最大10MB，保留5个备份
7. **前端主包**：有500kB+告警，不影响使用
8. **缓存**：单进程内存缓存，适合本地/单实例场景
9. **公众号文章**：默认保留30天，超期自动清理
10. **报告归档**：`reports/` 目录下按日期+类型命名，需定期清理
