## 1. 项目定位

`info-hub` 是一个以 `uwillberich` 方法论为核心的 **A 股交易工作台**。核心能力：

- **交易复盘**：按方法论输出市场分类、情景推演、持仓评估、时间门纪律
- **转强作战台**：盘前生成转强候选池，统一评估 `buy / watch / avoid`
- **复盘大师 Agent**：右侧常驻 AI 对话面板，全知项目数据，实时问答
- **投资日历**：未来 90 天重要会议/经济数据/政策日程，含受益板块与龙头

其余模块定位：

- `证据层`：财经新闻、热门板块、涨停分析、缠论图、概念作战图
- `内容层`：AI 新闻、热门话题、一键写文
- `前瞻层`：投资日历（新增）

判断顺序在代码里已被写死：

1. 市场分类（主线 / 独立龙头 / 区间防御）
2. 基准 / 乐观 / 风险情景
3. 时间门纪律
4. 用数据与新闻补证据

方法论约束入口：`backend/llm/uwillberich.py`

## 3. 核心流程怎么跑

### 3.1 交易复盘

前端入口：`frontend/src/components/review-report/ReviewPanel.tsx`

后端路径：

1. `ReviewPanel` 调用 `POST /api/review/generate`
2. `backend/routers/review_report.py` 拉取指数、板块、持仓行情
3. `backend/llm/prompts.py` 组装 prompt（六大维度结构化）
4. `backend/llm/qwen_client.py` 走 DashScope / Qwen 流式输出
5. 完整结果落表 `review_reports`
6. 草稿自动保存到 `review_draft`

### 3.2 转强作战台

前端入口：
- `frontend/src/components/turn-strong/TurnStrongPanel.tsx`
- `frontend/src/components/turn-strong/StrictTurnStrongPanel.tsx`

后端主逻辑：`backend/services/turn_strong_service.py`

执行顺序：

1. 先从 MX 接口做宽池筛选
2. 再补一层问财候选
3. 合并候选、去重、过滤 ST / 风险警示板
4. 补新闻、补实时行情、补多周期 K 线（问财接口）
5. 用 `uwillberich` prompt 做统一分析
6. 输出 `buy / watch / avoid`
7. 落表 `turn_strong_runs`
8. 次日用 `validation` 逻辑回算表现

### 3.3 复盘大师 Agent（新增）

前端入口：`frontend/src/components/assistant/AssistantPanel.tsx`（右侧常驻）

工作流程：

1. 用户输入问题 → `POST /api/assistant/chat` → 返回 `request_id`
2. 前端 `EventSource` 连接 `GET /api/assistant/stream/{request_id}`
3. 后台任务 `_execute_chat()` 并行执行：
   - 拉取持仓草稿、历史复盘、转强池、实时行情
   - 构建系统提示词（注入 uwillberich 方法论 + 实时数据）
   - SSE 流式推送 LLM 输出
4. 对话历史自动落表 `assistant_history`
5. "记住 xxx" 指令自动存入 `assistant_memories`

### 3.4 投资日历（新增）

前端入口：`frontend/src/components/investment-calendar/InvestmentCalendarPanel.tsx`

数据源：`backend/data/investment_calendar.py`（本地事件库，覆盖 4-12 月）

每个事件包含：日期、标题、类型、级别、描述、受益板块、龙头个股（含代码和推荐逻辑）

## 12. 当前已知问题

### 12.1 外部依赖重

只拷走本仓库无法独立运行，必须同时接住：
- `uwillberich` 桌面项目
- 本机环境变量（`.zshrc` / `.uwillberich/runtime.env`）
- DashScope key（Coding Plan 专用）
- iWenCai / MX key

### 12.2 K 线数据源切换

- AmazingData 已移除（服务器 1.8GB 内存不足，SDK 加载直接 OOM）
- 当前使用问财接口作为 K 线数据源，无需额外服务
- 分钟级 K 线暂不支持（问财只返回日线）

### 12.3 缓存仍是单进程内存缓存

涉及 `backend/services/market_service.py` 和 `turn_strong_service.py`，适合本地开发，不适合横向扩展。

### 12.4 调度器没有部署级隔离

无分布式锁，多实例部署会重复执行采集任务。

## 14. 后续开发优先级

1. **分钟级 K 线支持**：接入实时行情源（当前只有日线）
2. **复盘大师工具系统**：扩展为 ReAct Agent，支持调用行情查询/板块分析等工具
3. **投资日历数据源**：从静态文件升级为 API 拉取（金十数据/华尔街见闻）
4. **部署级改造**：日志持久化、守护进程、健康检查、配置文件化
5. **交易闭环增强**：转强验证统计页、失败样本归因、复盘模板对比

不建议重新往"全资讯平台"方向扩，当前代码已围绕 **交易复盘 + 转强作战台 + 复盘大师** 重组完成，再扩散会打散主轴。
