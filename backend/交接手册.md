# 分钟级K线支持升级

- **日期**: 2026-04-16
- **操作类型**: 功能增强
- **版本**: v2.1

## 背景

原系统问财接口只支持日线K线，所有周期（分钟/15分/小时）都映射为日线，前端K线矩阵无法展示分钟级图表。

## 变更内容

### 1. 数据源策略
- **分钟级K线**（1/5/15/30/60分钟）→ 东方财富免费接口
- **日线/周线/月线** → 问财接口（保持不变）

### 2. 修改文件
- `backend/services/quant_market_service.py`
  - 新增 `_fetch_eastmoney_kline()` 函数
  - 新增 `_eastmoney_secid()` 股票代码转换
  - 新增 `normalize_kline_period()` 周期标准化
  - 新增 `transform_kline_response()` 响应格式转换
  - 修改 `KLINE_PERIOD_ALIAS` 支持完整周期映射
  - 修改 `get_kline()` 分钟级走东财，日线走问财
  - 修改 `get_multi_period_klines()` 返回4个周期：minute/fifteen_minute/hour/day
  - 修改 `get_quant_capabilities()` 返回多数据源信息

- `backend/tests/test_quant_market_service.py`
  - 更新测试用例适配新架构
  - 新增 5分钟/30分钟周期测试
  - 新增四周期返回验证

### 3. 东方财富接口配置
- URL: `https://push2his.eastmoney.com/api/qt/stock/kline/get`
- 支持周期: min1(1), min5(2), min15(3), min30(4), min60(5), day(101), week(102), month(103)
- 无需API Key，免费无配额限制
- 数据格式: `"日期,开盘,收盘,最高,最低,成交量,成交额,..."`

## 测试结果

前端已有 `KlineMatrix.tsx` 组件支持4周期展示，无需修改前端代码。
API 返回结构保持一致，只是 `series` 中现在有真实数据。

# 交易闭环增强

- **日期**: 2026-04-16
- **操作类型**: 功能增强

原转强验证只有基础统计（成功数/失败数/平均涨跌幅），缺少成功率百分比、weak/flat 计数，以及失败样本归因分析。

## 变更内容

新增字段：
- `total_verifiable` - 可验证样本数（排除 insufficient）

- **冲高回落** - 盘中冲高但收盘大幅回落，承接力不足

# 复盘大师安全约束

- **日期**: 2026-04-16
- **操作类型**: 安全加固
- **版本**: v2.6

## 变更内容

### 1. 新增 `SAFETY_CONSTRAINTS` 安全约束层

**话题边界**：
- 只能讨论 A 股交易、市场分析、个股/板块研究、投资策略
- 无关话题必须拒绝并引导回交易相关

**禁止透露的信息**：
- 设备、服务器、操作系统、进程信息
- API Key、环境变量、配置文件路径、数据库路径
- 代码实现、技术架构、内部文件路径

**行为准则**：
- 明确定位为 A 股交易助手，不是通用聊天机器人
- 被问技术实现时统一回复标准话术

### 2. 注入位置
- `assistant_prompt.py` - 注入到复盘大师系统提示词**最前面**（最高优先级）
- `react_agent.py` - 注入到 ReAct Agent 系统提示词中

### 3. 文件变更

- `backend/services/react_agent.py` - 在 `SYSTEM_PROMPT_PREFIX` 中添加安全约束
- `backend/tests/test_safety_constraints.py` - 新增 7 个安全测试

# 复盘大师搜索能力（Agent-Reach）

- **日期**: 2026-04-16
- **操作类型**: 功能增强
- **版本**: v2.9

## 背景

复盘大师只能回答内置数据范围内的问题，无法获取最新新闻、公告、讨论等实时信息。

## 方案

基于 **Agent-Reach 项目底层能力（Jina Reader）** 实现股票搜索工具：
- Agent-Reach 是一个让 AI Agent 获得互联网搜索能力的工具
- 底层使用 Jina Reader (r.jina.ai) 读取任意网页内容
- 我们直接复用 Jina Reader API，无需安装完整 Agent-Reach 工具链

- 非股票查询会被拒绝："搜索内容必须与股票相关"

### 2. 安全约束
- 关键词必须包含 6 位股票代码 或 股票相关关键词（板块、行情、财报、涨停等）
- 非股票查询（如"今天天气"）直接拒绝
- 搜索结果限制 3000 字符，防止信息过载

### 3. 文件变更

  - 新增 3 个搜索工具测试

### 4. 工具列表（共 7 个）
| 工具名 | 功能 | 数据源 |
|--------|------|--------|
| query_stock_quote | 个股实时行情 | 腾讯行情 |
| query_index_quote | 大盘指数 | 腾讯行情 |
| query_kline | K 线数据 | 东财 + 问财 |
